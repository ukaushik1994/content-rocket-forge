## FIX 1: System Prompt Compression

**Issues:** #3, #5, #12

**Problem:** The system prompt now has 15+ sections injected every request. On GPT-4o-mini this is 15-20k tokens of instructions before any user data. The AI drowns in rules and TTFT stays at 20+ seconds.

### Backend

**File:** `supabase/functions/enhanced-ai-chat/index.ts`

The intent-gated module loading already exists. The problem is the ADDITIONAL sections added by the overhaul — response calibration, pushback, persona, workflow, feedback loop, trade-offs, content matching, service status, session checkpoint, workflow context — are ALL appended unconditionally.

Make the overhaul sections conditional too:

```ts
// Only add pushback protocol for write-intent queries

if (hasWriteIntent) {
  systemPrompt += PUSHBACK_PROTOCOL;
}

// Only add workflow orchestration for multi-step queries
const isMultiStep = /audit|pipeline|full|comprehensive|review all|plan for/i.test(userQuery);
if (isMultiStep) {
  systemPrompt += WORKFLOW_PROTOCOL;
}

// Only add trade-off reasoning for comparison queries
const isComparison = /should i|compare|which|better|vs|or /i.test(userQuery);
if (isComparison) {
  systemPrompt += TRADEOFF_PROTOCOL;
}

// Only add content matching for content-referencing queries
const referencesContent = /my article|my post|my content|the one about|that piece/i.test(userQuery);
if (referencesContent) {
  systemPrompt += CONTENT_MATCHING_PROTOCOL;
}

// Session checkpoint only every 8 messages (already conditional)
// Workflow context only when conversation has previous tool calls (already conditional)
// Service status only when unconfigured services exist (already conditional)
// Response calibration — ALWAYS include (it's the most impactful, only ~100 tokens)
// Persona — ALWAYS include (only ~50 tokens)

```

This reduces the average prompt by ~3000 tokens for simple queries.

## FIX 2: SEO Scoring Overhaul

**Issues:** #2, #7

**Problem:** Average SEO is 18/100. The `calculateBasicSeoScore` function is a 50-line heuristic that scores too harshly. Content may be well-structured but the scorer doesn't recognize it.

### Backend

**File:** `supabase/functions/enhanced-ai-chat/content-action-tools.ts` — replace `calculateBasicSeoScore` with a more balanced scorer:

```ts
function calculateBasicSeoScore(content: string, keyword: string, metaTitle?: string, metaDescription?: string): number {
  if (!content) return 0;
  let score = 0;
  const lowerContent = content.toLowerCase();
  const lowerKeyword = keyword?.toLowerCase() || '';
  const wordCount = content.split(/\s+/).length;

  // Content length (max 20 pts) — more generous thresholds
  if (wordCount >= 1000) score += 20;
  else if (wordCount >= 500) score += 15;
  else if (wordCount >= 300) score += 10;
  else score += 5;

  // Keyword in first 200 chars (10 pts)
  if (lowerKeyword && lowerContent.substring(0, 200).includes(lowerKeyword)) score += 10;
  else if (lowerKeyword && lowerContent.includes(lowerKeyword)) score += 5;

  // Keyword density 0.5-3% (10 pts)
  if (lowerKeyword) {
    const kwRegex = new RegExp(lowerKeyword.replace(/[.*+?^${}()|[\]\\]/g, '\\$&'), 'gi');
    const keywordCount = (lowerContent.match(kwRegex) || []).length;
    const density = (keywordCount / Math.max(wordCount, 1)) * 100;
    if (density >= 0.5 && density <= 3) score += 10;
    else if (density > 0) score += 5;
  }

  // Heading structure (max 20 pts)
  const h1Count = (content.match(/<h1|^# /gm) || []).length;
  const h2Count = (content.match(/<h2|^## /gm) || []).length;
  const h3Count = (content.match(/<h3|^### /gm) || []).length;
  if (h1Count >= 1) score += 5;
  if (h2Count >= 2) score += 10;
  else if (h2Count >= 1) score += 5;
  if (h3Count >= 1) score += 5;

  // Key Takeaways / TL;DR section (10 pts)
  if (/key takeaway|takeaway|tl;?dr|summary|highlights/i.test(content)) score += 10;

  // FAQ section (10 pts)
  if (/FAQ|frequently asked|common question/i.test(content)) score += 10;

  // Lists and structured content (10 pts)
  const listItems = (content.match(/<li|^[-*]\s|^\d+\.\s/gm) || []).length;
  if (listItems >= 5) score += 10;
  else if (listItems >= 2) score += 5;

  // Meta tags (10 pts)
  if (metaTitle && metaTitle.length >= 30 && metaTitle.length <= 60) score += 5;
  else if (metaTitle) score += 2;
  if (metaDescription && metaDescription.length >= 100 && metaDescription.length <= 160) score += 5;
  else if (metaDescription) score += 2;

  return Math.min(score, 100);
}
```

This new scorer gives more points for structure (Key Takeaways, FAQ, lists) which the enriched generation prompt actually produces. Maximum possible: 100. A well-structured AI-generated article should now score 60-80 instead of 15-25.

### Frontend

No changes. Score recalculates automatically on next content generation.

## FIX 10: AI Reuses Data From Earlier in Conversation

**Issues:** #31

**Problem:** Ask "show my content" → AI fetches data. 5 messages later ask "which should I publish?" → AI fetches the SAME data again.

### Backend

**File:** `supabase/functions/enhanced-ai-chat/tools.ts` — the tool caching already exists with `CACHE_TTL = 5 * 60 * 1000` (5 minutes). The issue might be that the cache is per-request, not per-conversation.

Check if the cache `Map` persists across tool calls within the same request:

The cache is created in `index.ts` and passed to `executeToolCall`. Within a single request with multiple tool calls, the cache works. But across separate user messages (separate edge function invocations), the cache is empty because edge functions are stateless.

**Fix:** Include recent tool results in the conversation context so the AI knows what was already fetched:

```ts
// In index.ts, before the AI call, check if recent messages contain tool results
const recentToolResults = messages
  .filter(m => m.role === 'assistant')
  .slice(-3)
  .filter(m => m.function_calls || m.visual_data);

if (recentToolResults.length > 0) {
  systemPrompt += `\n\n## RECENT DATA (already fetched — reuse instead of re-fetching):\nYou have already retrieved data in recent messages. If the user asks about the same data, reference your previous response instead of calling the tool again. Only call tools for NEW data the user hasn't seen.`;
}
```

This is a prompt instruction, not actual caching — but it's effective because the AI has the previous responses in its context and can reference them.
