# Smart Suggestions Plan — Kill Dumb Regex, Let AI Decide

> **Problem:** Suggestion buttons below AI responses are irrelevant — generated by client-side regex matching keywords in the response text, with zero understanding of context.
> **Solution:** Remove the regex system. Make the backend AI generate contextual follow-ups ONLY when it makes sense. Show nothing when there's nothing useful to suggest.

---

## THE FIX: 2 Changes

### Change 1: Backend — AI always considers generating follow-ups (but doesn't have to)

**File:** `supabase/functions/enhanced-ai-chat/index.ts`

Add to the system prompt (in the RESPONSE_STRUCTURE section or after it):

```
## FOLLOW-UP SUGGESTIONS (optional — include ONLY when genuinely useful)

After your response, you MAY include a "suggestedFollowUps" array with 2-3 short next-step prompts.

RULES:
1. Only include if there's a clear, valuable next step the user hasn't taken yet
2. Suggestions must be NEXT STEPS — not echoes of what was just discussed
3. If the response is a complete answer with no natural next step → DON'T include suggestedFollowUps
4. If the user asked a simple question → DON'T include suggestedFollowUps
5. Never suggest something the user just did or just asked about
6. Keep each suggestion under 8 words
7. Make them specific to THIS conversation, not generic

GOOD examples (contextual next steps):
- After showing keyword data: ["Write content for 'best CRM tools'", "Compare with competitor keywords"]
- After creating content: ["Optimize SEO score", "Schedule to calendar", "Repurpose for social"]
- After SERP analysis: ["Target low-difficulty keywords", "Create content brief"]

BAD examples (generic echoes):
- After discussing content: "Create a blog post about this topic" ← NEVER do this
- After keyword analysis: "Run a SERP analysis" ← user JUST did that
- After any response: "Show me charts" ← not specific, not useful

When you DO include them, format as:
"suggestedFollowUps": ["specific action 1", "specific action 2"]

When you DON'T (most responses), simply omit the field entirely.
```

**In the response parser** (where the AI response is parsed before sending to frontend):

Extract `suggestedFollowUps` from the AI response and include as `deepDivePrompts` in the `visualData`:

```ts
// After parsing AI response:
if (parsedResponse.suggestedFollowUps && parsedResponse.suggestedFollowUps.length > 0) {
  // Only include if AI generated them (not forced)
  if (!responseData.visualData) {
    responseData.visualData = {};
  }
  responseData.visualData.deepDivePrompts = parsedResponse.suggestedFollowUps;
}
```

## WHAT CHANGES FOR THE USER

### Before (current):
- Every AI response shows 3 suggestion buttons
- Buttons are generic regex matches ("Create a blog post", "Run SERP analysis")
- Buttons often repeat what was just done
- No way to turn them off
- Feels spammy and irrelevant

### After (new):
- Most AI responses show NO suggestion buttons (clean, complete answers)
- When AI HAS a genuine next step → 2-3 specific, contextual suggestions appear
- Suggestions are actual next steps: "Schedule this to calendar", "Write content for 'best CRM tools'"
- AI decides — if the answer is complete, nothing shows
- Feels intelligent and intentional

## WHY THIS IS GENIUS

1. **No extra API cost** — the AI generates follow-ups as part of its response. No separate call.
2. **Naturally contextual** — the AI knows what it just said, what tools it called, what data it showed. Its suggestions will be relevant.
3. **Self-regulating** — the prompt tells AI to OMIT suggestions when unnecessary. Less noise, more signal.
4. **No maintenance** — no regex patterns to update when new tools are added. AI learns tool names from its tool definitions.
5. **Graceful degradation** — if AI doesn't include suggestions, nothing shows. Better than wrong buttons.
